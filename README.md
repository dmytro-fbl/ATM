# ATM System (Система управління банкоматом)

[![.NET](https://img.shields.io/badge/.NET-8.0%2F9.0-blueviolet)](https://dotnet.microsoft.com/)
[![EF Core](https://img.shields.io/badge/ORM-EF%20Core-blue)](https://learn.microsoft.com/en-us/ef/core/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

Проєкт представляє собою повнофункціональну **Full-stack систему** (Web API + Frontend), що імітує роботу програмного забезпечення банкомата. Додаток складається з потужної серверної частини на ASP.NET Core та інтерактивного клієнтського інтерфейсу, що забезпечує повний цикл взаємодії користувача з банківською системою.

---

## Основні можливості

* **Аутентифікація та RBAC:** Безпечний вхід за номером картки та ПІН-кодом (з використанням хешування). Розділення прав доступу на "Клієнт" та "Адміністратор" (Role-Based Access Control).
* **Управління готівкою:** Розумний алгоритм зняття та поповнення з автоматичним оновленням касет банкомата.
* **Термінал адміністратора:** Окрема консоль для моніторингу залишку купюр та проведення інкасації (поповнення касет) в режимі реального часу.
* **Сувора валідація:** Перевірка валідності номерів карток за алгоритмом Луна та контроль жорстких лімітів на суми транзакцій.
* **Оптимізована історія:** Виведення історії транзакцій користувача з підтримкою пагінації (посторінкового виводу) для зниження навантаження на БД.
* **Транзакційність:** Використання ACID-транзакцій для гарантії збереження коштів при збоях.
* **Глобальна обробка помилок:** Middleware для перехоплення виключень та повернення чистих JSON-відповідей.
* **Повний аудит:** Логування бізнес-подій та технічних помилок у базу даних.

## Принципи та Патерни

### Programming Principles
* **SRP (Single Responsibility):** Розділення відповідальності між контролерами, сервісами, репозиторіями та класами валідації.
* **DIP (Dependency Inversion):** Високорівневі модулі залежать від інтерфейсів, а не від реалізацій.
* **DRY (Don't Repeat Yourself):** Спільна логіка винесена в приватні хелпер-методи.
* **Fail Fast:** Система миттєво викидає Exception або 400 Bad Request при невалідних даних на найраніших етапах.

### Design Patterns
* **Repository Pattern:** Ізоляція EF Core від бізнес-логіки.
* **Service Layer:** Центральна ланка (`AtmService`), яка координує роботу репозиторіїв.
* **DTO (Data Transfer Object):** Використання ізольованих об'єктів (напр. `LoginRequest`, `RefillRequest`) для безпечної передачі даних між Frontend та Backend.
* **Middleware Pattern:** Глобальний обробник помилок `ExceptionHandlingMiddleware`.
* **Strategy Pattern:** Реалізовано в алгоритмах видачі готівки (`ICashWithdrawalStrategy`), що дозволяє легко змінювати спосіб підбору купюр. (В майбутньому буде реалізовано метод видачі великими або малими купюрами).
* **Template Method:** Використано абстрактний `BaseService` для створення єдиного шаблону логування подій у всіх сервісах системи.
* **Singleton:** Застосовано для сервісу хешування паролів (`IPasswordHasher`), забезпечуючи єдиний екземпляр об'єкта протягом роботи додатка.

## Безпека та Аудит

### Logging System
Система веде журнал у таблиці `AtmOperationLogs`:
* **Info:** Успішні операції, входи в систему та дії адміністратора (інкасація).
* **Warning:** Невдалі спроби входу, порушення лімітів.
* **Error:** Критичні помилки сервера із записом **StackTrace**.

### Exception Handling
Реалізовано Middleware, який повертає помилки у форматі:
```json
{
  "error": "Текст помилки для користувача"
}
```
