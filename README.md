# ATM System (Система управління банкоматом)

Проєкт представляє собою Backend-частину (Web API) для програмного забезпечення банкомата. Система дозволяє керувати рахунками, обробляти транзакції, видавати готівку з урахуванням наявності купюр у касетах та вести аудит усіх операцій.

Programming Principles (Принципи програмування)
У цьому проєкті дотримано наступних принципів:

SRP (Single Responsibility Principle): Кожен клас має лише одну зону відповідальності. Наприклад, Transaction відповідає лише за структуру даних, а TransactionRepository - виключно за запити до бази даних.

DIP (Dependency Inversion Principle): Високорівневі модулі (контролери/сервіси) не залежать від низькорівневих (база даних). Замість цього вони залежать від абстракцій (інтерфейсів, таких як IAtmCassetteRepository).

DRY (Don't Repeat Yourself): Логіка роботи з даними винесена в репозиторії, що дозволяє уникати дублювання LINQ-запитів у різних частинах програми.

KISS (Keep It Simple, Stupid): (В процесі: буде реалізовано в бізнес-логіці сервісів)

Fail Fast: (В процесі: буде реалізовано у валідації транзакцій)

Design Patterns (Патерни проєктування)
Проєкт використовує такі патерни:

Repository Pattern: Реалізовано для всіх сутностей (TransactionRepository, AtmCassetteRepository тощо). Дозволяє ізолювати Entity Framework Core від бізнес-логіки.

Dependency Injection (DI): Всі залежності (контекст БД, репозиторії) реєструються в Program.cs та інжектяться через конструктори.

Strategy Pattern: (Заплановано для алгоритму розрахунку видачі купюр різного номіналу).

Refactoring Techniques (Техніки рефакторингу)
Під час розробки застосовувалися такі техніки:

Extract Interface: Виділення абстракцій (наприклад, IAtmOperationLogRepository) з класів репозиторіїв для підтримки DIP та полегшення майбутнього тестування.

Change Value to Reference / Refine Types: Перехід від стандартних типів до Guid для підвищення безпеки ідентифікаторів, а також використання Nullable типів (Guid? CardId) для гнучкості логування незалежних подій.

Rename Variable / Method: Приведення назв властивостей та методів до єдиного стандарту для покращення читабельності коду.

Guard Clauses: (Заплановано для обробки помилок пін-кодів та балансу)

Extract Method: (Заплановано для декомпозиції складних алгоритмів підрахунку грошей)
